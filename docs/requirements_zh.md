# ZakoBox/ZakoPako 项目需求文档

## 1. 项目概述

### 1.1 项目背景
#### 1.1.1 项目起源
- 为 Project Airi 的 Core-Dev-DAO，即 ZakoDAO 开发的专属链上治理和募资工具
- 解决开发者分散的开源项目在团队协作、资金管理和决策治理方面的痛点
- 作为 ETHOnline 2025 黑客松参赛项目（10 月 26 日项目提交截止）

#### 1.1.2 项目命名
- ZakoBox：面向英语母语用户的简单直观命名
- ZakoPako：面向二次元开发者社区的命名（zako + hako 箱的浊化音）
- 统一的像素极简风格 Logo 设计（TODO）

### 1.2 核心定位
#### 1.2.1 产品定位
- 链上版本的 OpenCollective（黑客松上开发的 MVP 的主要功能）
- Guild.xyz、SAFE 前端和 Snapshot/Aragon 的功能集合体
- 专注于开发团队内部治理，而非大规模社区治理

#### 1.2.2 目标用户
- 开源项目核心开发团队
- 需要链上募资的技术团队
- 寻求透明资金管理的 DAO 组织

#### 1.2.3 独特价值
- 一站式解决方案：DAO 创建、管理、投票、多签金库维护
- 与 GitHub 无缝集成的链上募资渠道+稳定币/信用卡支付网关
- 完全由开源项目开发团队掌控的链上募资平台

## 2. 功能需求

### 2.1 DAO 管理功能（黑客松 MVP 暂不涉及，可在前端预留 Placeholder）
#### 2.1.1 DAO 创建
- 一键创建 DAO 组织
- 自定义 DAO 参数设置（名称、描述、Logo）
- 初始成员邀请机制
- DAO 模板选择（开发团队型、社区型、混合型）

#### 2.1.2 团队管理
- 核心开发团队成员管理
- 权限分级管理（创始人、核心开发者、贡献者）
- 基于角色的访问控制（RBAC）
- 成员贡献度追踪

#### 2.1.3 多签配置
- 多签钱包自动部署
- 灵活的签名阈值设置（推荐3/5）
- 签名者权重配置
- 紧急情况处理机制

### 2.2 投票治理功能（黑客松 MVP 暂不涉及，可在前端预留 Placeholder）
#### 2.2.1 提案系统
- 提案模板库（技术决策、资金分配、人事变动）
- 提案创建向导
- 提案讨论区（链上评论）
- 提案状态管理（草案、投票中、已通过、已执行）

#### 2.2.2 投票机制
- 链上投票记录永久保存
- 多种投票类型（简单多数、超级多数、加权投票）
- 投票权重配置（基于贡献度、持币量、时间锁定）
- 投票委托机制

#### 2.2.3 执行机制
- 投票结果自动执行
- 时间锁定期设置
- 执行失败回滚机制
- 执行日志记录

### 2.3 金库管理功能
#### 2.3.1 ZakoBox 智能合约架构
- ZakoBox 金库合约（资金存储和管理）
- ZakoBoxFactory 工厂合约（批量部署 ZakoBox 合约）
- 预设解锁期和资金流向配置
- 多 Token 支持

#### 2.3.2 资金管理
- 实时金库余额展示
- 资金分配规则引擎
- 自动化资金释放（基于里程碑）
- 紧急提取机制（基于多签，需要规定合适的 Threshold 数量）

#### 2.3.3 资金安全
- 多重签名保护
- 每日提取限额
- 白名单地址管理
- 异常交易监控

### 2.4 链上募资功能
#### 2.4.1 募资渠道
- GitHub 集成（一键从 GitHub 捐赠到指定的 ZakoBox 金库）
- 直接链上捐赠入口
- 众筹活动管理（目标金额、截止时间、奖励设置）
- 定期订阅捐赠

#### 2.4.2 支付方式
- PYUSD 稳定币优先集成（黑客松奖励点）
- 主流 Token 支持（ETH、USDC、USDT）
- 跨链资产桥接（集成 Jumper 等跨链桥聚合器）

#### 2.4.3 捐赠管理
- 捐赠者身份验证（可选 KYC）
- 捐赠证明 NFT 发放
- 税务报表生成
- 退款机制

### 2.5 数据透明化功能
#### 2.5.1 链上数据展示
- Blockscout SDK 深度集成（黑客松奖励点）
- 简化的捐赠/众筹数据仪表板
- 实时金库余额和资金流向
- 历史数据分析图表

#### 2.5.2 交易追踪
- 每笔交易详细记录
- 资金流向可视化（Sankey 图）
- 捐赠者贡献排行榜
- 支出分类统计

#### 2.5.3 报告生成（黑客松 MVP 暂不涉及，可在前端预留 Placeholder）
- 月度/季度财务报告
- 项目进度报告
- 捐赠者感谢信自动生成
- 合规性报告

### 2.6 资本效率优化
#### 2.6.1 质押服务集成
- 闲置资金自动质押
- 多种质押协议选择（AAVE、Compound）
- 收益自动复投
- 风险评级和预警

#### 2.6.2 动态贪婪模型（黑客松 MVP 暂不涉及，可在前端预留 Placeholder）

- 防止大户操纵算法
- 套利保护机制
- 流动性激励

#### 2.6.3 资金利用分析（黑客松 MVP 暂不涉及，可在前端预留 Placeholder）
- 资金使用效率评分
- 优化建议生成
- 对标分析（与其他 DAO 对比）
- 预算规划工具

## 3. 技术需求

### 3.1 智能合约
#### 3.1.1 开发环境
- Hardhat 3.0.0+ 开发框架
- Solidity 0.8.20+
- OpenZeppelin合约库
- 多链部署支持

#### 3.1.2 合约架构
- 可升级合约设计（UUPS 代理模式）
- 模块化合约结构
- Gas 优化策略
- 事件日志完善

#### 3.1.3 测试要求
- 单元测试覆盖率 >90%
- 集成测试完整性
- 模糊测试（Fuzzing）
- 形式化验证（关键合约）

### 3.2 前端 / UI
#### 3.2.1 技术栈
- TypeScript：核心语言
- Vue：核心前端框架
- Vite：打包工具
- Vue-router：路由
- Vue-i18n：国际化与本地化
- unocss：样式
- pnia：状态管理
- shadcn：UI 库
- pnpm：包管理器
- oxlint/eslint：代码风格和质量保证
- GitHub Actions：持续集成/持续部署
- VueUse：工具库

#### 3.2.2 用户体验
- 响应式设计（移动端优先）
- 多语言支持（英语、中文）
- 暗色/亮色模式
- 无障碍设计（WCAG 2.1 AA 级）（可选）

#### 3.2.3 Web3集成
- 多钱包支持（MetaMask、WalletConnect、Coinbase Wallet）
- 网络自动切换
- 交易状态实时追踪
- 签名请求优化

### 3.3 后端服务 （如有需要）
#### 3.3.1 API服务
- Node.js + Express/Fastify
- GraphQL API
- WebSocket 实时通信
- 速率限制和DDoS防护

#### 3.3.2 数据存储
- IPFS 去中心化存储
- PostgreSQL 关系数据库
- Redis缓存层
- 事件溯源架构

#### 3.3.3 第三方集成
- GitHub API（仓库信息、贡献者数据）
- Blockscout API（链上数据查询）
- PYUSD支付网关
- 消息推送服务

### 3.4 安全需求
#### 3.4.1 智能合约安全
- 重入攻击防护
- 整数溢出检查
- 权限控制严格
- 紧急暂停机制

#### 3.4.2 应用安全
- DID-based KYC 系统
- 防女巫攻击机制
- 输入验证和消毒
- CORS 和 CSP 配置

#### 3.4.3 审计要求
- 智能合约第三方审计（AI-based 自我审计）
- 渗透测试
- 代码静态分析
- 依赖项安全扫描

- 完成 MVP 核心功能
- 通过所有测试用例
- 获得至少一个奖项

### 8.2 产品发布阶段
- 首月活跃 DAO > 10 个
- 月度交易量 > $100,000
- 用户满意度 > 4.5/5
- 零安全事故

### 8.3 长期目标
- 内部版本：成为 ZakoDAO 专属的募资平台并被有效使用
- 社区版本：被 > 50 个开源项目团队自部署
- 商业版本：服务 > 50 个开源项目团队
- 商业版本：年度募资额 > $10M
- 商业版本：建立可持续商业模式
